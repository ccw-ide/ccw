Setting Up Dependencies to Clojure

Creating a Clojure project or adding Clojure support to an existing Java project will automatically add a dependency to the Clojure Plugin's own embedded versions of Clojure and clojure-contrib, if the Clojure JAR file is not found in the classpath.

Clojure itself must always be a dependency of the project, but this is highly customizable. It can come in as many flavors as Eclipse allows:

  • Clojure source code directly embedded in the project (not recommended)
  • As another project in the Eclipse workspace. You then make the Clojure project a dependency of your own project (recommended if you want to stay in sync with Clojure's development):
    • First create a classic java project for the source code of Clojure. You will have to change the default Java source folder: You must have two of them, one will be src/jvm, the other will be src/clj.
    • In a second project where you enabled Clojure support, do Project > Properties > Java Build Path > Projects > Add and add the Clojure project.
  • As a JAR file. Use the classical Java mechanism offered by Eclipse to add the JAR file to your project (Project > Properties > Java build path > Libraries ...).
